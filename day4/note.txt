[Thread] - 코드를 관통하는 실이다.
1. 쓰레드 프로그래밍은 객체지향과 거리가 멀다.
2. 객체지향을 먼저 작성하고 쓰레드를 적용한다.

extends Thread - 객체 하나당 실 하나를 만들 때 사용한다.*
implements Runable - 객체는 하난데 실이 여러개 지나가야할 때 사용한다.*
inner Class ( 람다식 )

멀티 쓰레드 - 동시성, 병렬

자바에서 가장 메모리에 무거운 동작은 Thread 생성하는 행위이며, 그다음이 새로운 객체를 생성하는 행위이다.

객체는 하나이고 쓰레드를 여러개 돌리는 방식의 코드를 작성한다.

Thread.sleep() - CPU를 점유하지 않는 시간을 확정한다. 하지만 그 이후에 언제 점유할지에 대한 부분은 확정적이디 않다.

Java에서 클로저는 반쪽짜리 클로저이다**

클로저는 숨겨져 있는 참조
함수 자체에는 핻아 변수의 값을 선언하거나 넘겨받지 않아도, 외부의 변수의 값을 참조하고 있는 것

JavaScript에서는 람다식에서 외부 변수의 값을 변경할 수 있다.
Java 에서는 this참조가 불가능하며, 외부에서 참조하는 변수의 변경이 불가능하다.
